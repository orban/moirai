## What we're building

A 5-panel illustrated explainer as a Hugo blog post on ryanorban.com. Single-page scrollytelling format inspired by jalammar.github.io/illustrated-transformer/. Inline SVG figures generated from real moirai data. Light theme matching the Hugo blog.

Sequel to the existing post "Stop Testing AI Agents Like Deterministic Code" — that post argues you should treat agents as stochastic processes. This one shows what you find when you do.

## Why this approach

- Hugo blog already exists with a published related post
- Single HTML with inline SVGs is portable (share ahead of Monday calls, lives permanently on the blog)
- 5 panels is buildable in a session, expandable later to the full 12-panel illustrated guide
- The visual format makes the methodology intuitive to non-researchers (hiring managers, product teams)

## The 5 panels

### Panel 1: Same agent, different outcomes
10 horizontal trajectory bars for one task. Each bar is a sequence of colored step blocks. 5 green-bordered (pass), 5 red-bordered (fail). The reader sees identical starting conditions, different outcomes.

Data source: one SWE-rebench task with mixed outcomes (e.g., whatshap__whatshap-412, 15P/14F).

### Panel 2: Align them
The 10 bars snap into columns via NW alignment. Matching steps line up vertically. Gaps appear where runs diverge. Animated or side-by-side before/after showing raw sequences → aligned matrix.

Data source: alignment matrix from moirai's align_runs on the same task.

### Panel 3: Find the fork
Zoom into the divergence column. Left side: what pass runs did (with actual reasoning text). Right side: what fail runs did (with actual reasoning text). The reader sees the specific decision that determined the outcome.

Data source: one pairwise divergence from extract_pairwise_preferences.

### Panel 4: Scale it up — failure modes
Pull back to show all 1,096 mixed-outcome tasks. Each divergence point is a dot. Cluster them. Show the treemap: 25% wasted orientation, 18% wrong file, 13% skipped reasoning, etc.

Data source: divergence_clusters output on SWE-rebench data.

### Panel 5: What predicts success
Forest plot of the 4 robust features (test_position_centroid, symmetry_index, reasoning_hypothesis_formation, edit_concentration). Each with effect size, confidence direction, and one-line interpretation. End with: "These are within-task predictive effects from 12,854 same-agent runs. The methodology and code are open source."

Data source: swarm results + split-half validation.

## Key decisions

- **Hugo blog post**, not standalone HTML — lives on ryanorban.com alongside the existing agent testing post
- **5 panels for v1** — expandable to 8-12 later with Simpson's paradox, feature correction, swarm, DPO
- **Real data in every figure** — no synthetic illustrations, everything generated from actual moirai output
- **SVG figures generated by Python scripts** — not hand-drawn, reproducible from the data
- **Light theme** matching the Hugo blog's existing design (ryanorban.com uses hugo-theme-til, light background)
